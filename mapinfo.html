  • Limitations

  • Projection

    Currently, the only projection accepted is the default one: EPSG 3857. All data must be in this projection for it to work in the application.

  • Shapefiles

    Native JavaScript implementations for reading shapefiles are crap and work too slow. If you have a shapefile within ArcMap or QGIS you can export the feature as a JSON to work with Leaflet Map Assist.

  • Data Size

    Without the use of a server, the resulting map and all the assets will be downloaded on the load of the site. Therefore pruning data and properties of data can benefit the map performance and load time. Also, loading super large datasets will crash the application and make it impossible to export. For large datasets a server is necessary.

  • Layer Name Sensitivity

    Currently, adding layers with same name is not allowed.

  • Local File Loading

    Data parser is using a local file i/o to load the data, in the future we will allow for using links to load data from APIs with Fetch API.

  • Styling layers

    The application is at early stages of development, but will be coming in later revisions.

  • Additional Info:

    For tooltips and popup forms you can add layer property variables. ie. A dataset with property of species (layer.properties.species) can be used by adding dollar sign and property name -> $species. When using $ symbol for currency add an extra $ symbol as it wil escape and return $ symbol and the currency value instead of trying to find property with name of currency value.
